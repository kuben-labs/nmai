# AGENTS.md — NorgesGruppen Grocery Detection Competition

## Project Overview

Object detection competition: detect and classify grocery products on store shelves.
Scoring: `0.7 * detection_mAP@0.5 + 0.3 * classification_mAP@0.5` (max 1.0).
Detection-only (all `category_id: 0`) caps at 0.70.

## Environment & Commands

### Training Environment Setup
```bash
pip install ultralytics==8.1.0
pip install torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cu124
pip install pycocotools==2.0.7 albumentations==1.3.1
pip install ruff  # dev only
```

### Data Preparation
```bash
python scripts/coco_to_yolo.py  # uses defaults, creates data/yolo/
```

### Training
```bash
# Quick baseline (~30 min on A100)
python scripts/train.py --model yolov8m.pt --imgsz 640 --epochs 50 --name baseline

# Full training (recommended)
python scripts/train.py --model yolov8m.pt --imgsz 1280 --epochs 150 --name full

# Scale up for best accuracy
python scripts/train.py --model yolov8l.pt --imgsz 1280 --epochs 150 --name large

# Resume interrupted training
python scripts/train.py --resume runs/full/weights/last.pt
```

### Validation & Evaluation
```bash
# YOLO built-in validation
yolo detect val data=configs/dataset.yaml model=runs/full/weights/best.pt imgsz=1280

# Single-image prediction
yolo detect predict model=runs/full/weights/best.pt source=data/raw/train/images/img_00001.jpg

# Competition score (local, on val split)
python scripts/evaluate.py --model runs/full/weights/best.pt \
    --images data/yolo/images/val --val-only

# Competition score from predictions file
python scripts/evaluate.py --predictions output/predictions.json --val-only
```

### Testing & Submission
```bash
# Test inference locally (mirrors sandbox)
python scripts/run.py --input data/raw/train/images --output output/predictions.json

# Validate predictions format
python scripts/validate_predictions.py --predictions output/predictions.json \
    --annotations data/raw/train/annotations.json

# Package submission zip
python scripts/package_submission.py --weights runs/full/weights/best.pt

# Verify zip structure
unzip -l submission.zip | head -10
```

## Code Style Guidelines

### Python Version & Imports
- Target Python 3.11 (sandbox runtime)
- Use `pathlib.Path` instead of `os.path` — `os` is blocked in the sandbox
- Use `json` instead of `yaml` — yaml is blocked in the sandbox
- Never import: `os, sys, subprocess, socket, ctypes, builtins, importlib,
  pickle, marshal, shelve, shutil, yaml, requests, urllib, http.client,
  multiprocessing, threading, signal, gc, code, codeop, pty`
- Never call: `eval(), exec(), compile(), __import__(), getattr()` with dangerous names
- Group imports: stdlib, third-party, local — separated by blank lines

### Formatting
- `ruff` for formatting and linting; 4-space indent; max 100 char lines
- Trailing commas in multi-line collections; double quotes for strings

### Types & Naming
- Type hints on function signatures
- snake_case for functions/variables, PascalCase for classes, UPPER_CASE for constants
- Descriptive names: `predictions` not `preds`, `image_path` not `img_p`

### Error Handling
- Explicit try/except with specific exceptions; log errors with context
- Fail fast during data loading, gracefully during inference

### Model Code Conventions
- Auto-detect GPU: `device = "cuda" if torch.cuda.is_available() else "cpu"`
- `torch.no_grad()` during inference; FP16 on GPU; process images one at a time
- State dict saves only: `torch.save(model.state_dict(), ...)` not `torch.save(model, ...)`

### run.py Contract
```python
# python run.py --input /data/images --output /output/predictions.json
# Output: JSON array of {"image_id": int, "category_id": int, "bbox": [x,y,w,h], "score": float}
# bbox = COCO format [x, y, width, height]; image_id from filename: img_00042.jpg → 42
```

## Data Notes

- 248 training images, 22,731 annotations, 356 categories (IDs 0-355)
- Category 355 = "unknown_product" (422 annotations, most frequent)
- Images: 481-5712px wide, 399-4624px tall; avg 92 annotations/image
- Product reference images: 327 products with multi-angle shots in `data/raw/NM_NGD_product_images/`
- YOLO split: 211 train / 37 val (seed=42, 15% val)
